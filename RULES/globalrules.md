GLOBAL MUST-HAVE INSTRUCTIONS

Before writing any code:

* Read the entire memory bank/project context first
* Understand the current architecture
* Reuse existing components and patterns
* Never create duplicate UI patterns
* Maintain strict UI consistency
* Follow the existing design system
* Keep components modular and reusable
* Avoid overengineering
* Avoid unnecessary dependencies
* Maintain scalable folder structure
* Use clean architecture principles
* Prioritize readability and maintainability

UI RULES:

* Use shadcn/ui consistently
* Maintain consistent spacing, typography and colors
* Keep layouts clean and minimal
* Avoid cluttered dashboards
* Use responsive design everywhere
* Include loading, empty and error states
* Use reusable cards, dialogs, forms and tables
* Keep animations subtle and premium

CODE RULES:

* Strong TypeScript typing
* No hardcoded values when avoidable
* Use environment variables properly
* Use reusable hooks/services/utils
* Separate business logic from UI
* Keep files small and maintainable
* Use proper naming conventions
* Avoid spaghetti code

BACKEND RULES:

* Maintain modular FastAPI architecture
* Separate routes/services/repositories/models
* Use async where appropriate
* Write scalable database schemas
* Use proper validation
* Prepare APIs for future integrations

WORKFLOW RULES:

* Split work into phases
* Complete one phase before moving forward
* At the end of each phase:

  * summarize completed work
  * explain architecture decisions
  * validate consistency
  * list next steps

DO NOT:

* randomly refactor working systems
* create inconsistent UI
* introduce breaking architectural changes
* generate unnecessary complexity
* duplicate components or logic

Always optimize for:

* scalability
* maintainability
* developer experience
* clean UX
* production readiness
